# 符数计算规则

本文档记录日麻符数 `m` 的计算规则。代码实现以这里的规则为准。

## 牌编码约定

| 编码 | 含义 |
| --- | --- |
| `1m` 到 `9m` | 一万到九万 |
| `1p` 到 `9p` | 一筒到九筒 |
| `1s` 到 `9s` | 一索到九索 |
| `1z` | 东风 |
| `2z` | 南风 |
| `3z` | 西风 |
| `4z` | 北风 |
| `5z` | 白板 |
| `6z` | 发 |
| `7z` | 中 |

## 计算流程

1. 先判断特殊牌型。
2. 非特殊牌型从底符 `20` 开始累加。
3. 累加雀头、刻子、杠子、等待形和和牌方式符。
4. 除七对子固定 `25` 符外，其余结果向上取整到十位。

## 特殊牌型

| 牌型 | 符数 | 说明 |
| --- | ---: | --- |
| 七对子 | 25 | 固定 `25` 符，不再取整。 |
| 平和自摸 | 20 | 固定 `20` 符。 |
| 副露平和形 | 30 | 副露、四顺子、非役牌雀头、无单骑/边张/嵌张加符时固定 `30` 符。 |

## 基础符

普通牌型统一底符：

| 项目 | 符数 |
| --- | ---: |
| 底符 | 20 |

## 雀头符

役牌雀头按满足的条件累加。

| 雀头类型 | 符数 |
| --- | ---: |
| 三元牌雀头（白、发、中） | +2 |
| 自风雀头 | +2 |
| 场风雀头 | +2 |

如果同一风牌同时是自风和场风，则合计 `+4`。

## 刻子和杠子符

刻子和杠子先按明面子取基础值，再根据暗面子和幺九牌/字牌翻倍。

| 面子 | 中张牌 | 幺九牌/字牌 |
| --- | ---: | ---: |
| 明刻 | +2 | +4 |
| 暗刻 | +4 | +8 |
| 明杠 | +8 | +16 |
| 暗杠 | +16 | +32 |

说明：

- 中张牌指 `2` 到 `8` 的数牌。
- 幺九牌/字牌指 `1`、`9` 和所有字牌。
- 顺子不加符。

## 等待形符

| 等待形 | 符数 | 说明 |
| --- | ---: | --- |
| 单骑 | +2 | 和牌张组成雀头。 |
| 边张 | +2 | `1-2-3` 听 `3`，或 `7-8-9` 听 `7`。 |
| 嵌张 | +2 | 顺子中间张听牌。 |
| 两面 | +0 | 不加符。 |

## 和牌方式符

| 和牌方式 | 符数 |
| --- | ---: |
| 自摸 | +2 |
| 门前清荣和 | +10 |

平和自摸属于特殊牌型，固定 `20` 符，不额外计算自摸 `+2`。

## 取整规则

| 情况 | 规则 |
| --- | --- |
| 七对子 | 固定 `25` 符，不取整。 |
| 其他牌型 | 原始符数向上取整到十位。 |

示例：

| 原始符数 | 最终符数 |
| ---: | ---: |
| 20 | 20 |
| 22 | 30 |
| 30 | 30 |
| 32 | 40 |

## 当前实现备注

- 符数计算入口是 `DefaultFuCalculator`。
- 手牌拆分结果需要提供面子类型：顺子、刻子、杠子。
- 明暗面子优先使用面子上的 `IsOpen` 标记；上下文中的 `IsOpenHand` 用于判断整手是否副露。
- 杠子只有在拆分结果中标记为 `Quad` 时才按杠子计算。
- 等待形根据和牌张 `WinningTile` 和当前拆分结果推断；如果和牌张未知，则不计算等待形符。
- 荣和完成的刻子按明刻计算；自摸完成的刻子按暗刻计算。
