# 役种规则

本文档用于记录日麻役种规则。后续代码实现以这里写清楚的规则为准。

## 维护约定

- 每个役种单独写一节。
- 如果规则还不确定，写 `待确认`，不要模糊带过。
- 牌编码统一使用本文档的编码约定。
- 判断条件尽量写成代码可以直接理解的条件。
- 涉及和牌方式、是否门清、是否副露、是否立直等状态时，写在“需要上下文”里。

## 牌编码约定

| 编码 | 含义 |
| --- | --- |
| `1m` 到 `9m` | 一万到九万 |
| `1p` 到 `9p` | 一筒到九筒 |
| `1s` 到 `9s` | 一索到九索 |
| `1z` | 东风 |
| `2z` | 南风 |
| `3z` | 西风 |
| `4z` | 北风 |
| `5z` | 白板 |
| `6z` | 发 |
| `7z` | 中 |

面子类型：

| 类型 | 含义 | 示例 |
| --- | --- | --- |
| 顺子 | 同花色连续三张数牌 | `2m 3m 4m` |
| 刻子 | 三张相同牌 | `5z 5z 5z` |
| 杠子 | 四张相同牌 | `5z 5z 5z 5z` |
| 雀头 | 两张相同牌 | `1z 1z` |

## 役种模板

复制下面模板来新增役种。

```markdown

- ID：
- 中文名：
- 拼音：
- 番数：
- 副露后番数：
- 是否门前限定：
- 适用牌型：
- 判断条件：
- 不成立条件：
- 可复合役种：
- 互斥役种：
- 需要上下文：
- 实现提示：
- 测试牌例：
- 备注：
```

字段说明：

| 字段 | 写法 |
| --- | --- |
| ID | 代码里使用的稳定标识，例如 `duanyaojiu`、`yipai-baiban` |
| 拼音 | 中文名的拼音，用于后续代码命名或显示，例如 `duan yao jiu` |
| 番数 | 门前时的番数 |
| 副露后番数 | 副露后如果不能成立，写 `不成立` |
| 是否门前限定 | `是` 或 `否` |
| 适用牌型 | 标准型、七对子、国士无双、特殊形等 |
| 判断条件 | 役种成立的核心规则 |
| 不成立条件 | 容易误判的排除条件 |
| 可复合役种 | 可以同时计算的役种，需要全部写出具体名称；多个役种用 `、` 分割 |
| 互斥役种 | 成立后不能同时算的役种 |
| 需要上下文 | 是否自摸、荣和、立直、场风、自风、宝牌等 |
| 实现提示 | 给代码实现看的注意点 |
| 测试牌例 | 至少写一个成立例子，最好再写一个不成立例子 |
