### 2026-06-07：DockPanel 是什么，WPF 里面有吗，具体是什么效果？

**问题**

DockPanel 是什么，WPF 里面有吗，具体是什么效果？

**知识点**

DockPanel 是一种布局容器，用来把子控件停靠到父容器的上、下、左、右四个边缘。它适合做工具栏、状态栏、侧边栏、顶部导航等典型布局。

WPF 里有 DockPanel。Avalonia 里也有 DockPanel，概念和用法基本一致。

DockPanel 的核心属性是 `DockPanel.Dock`，写在子控件上：

```xml
<DockPanel>
    <Button DockPanel.Dock="Top" Content="顶部工具栏"/>
    <Button DockPanel.Dock="Left" Content="左侧菜单"/>
    <TextBlock Text="剩余内容区域"/>
</DockPanel>
```

效果是：

- `DockPanel.Dock="Top"` 的控件贴在顶部。
- `DockPanel.Dock="Bottom"` 的控件贴在底部。
- `DockPanel.Dock="Left"` 的控件贴在左边。
- `DockPanel.Dock="Right"` 的控件贴在右边。
- 默认最后一个子控件会填满剩余空间。

DockPanel 有一个重要属性：`LastChildFill`。默认通常为 `True`，表示最后一个子控件不再按 Dock 方向摆放，而是自动占满剩余区域。如果设置为 `False`，最后一个子控件也会按自己的 `DockPanel.Dock` 正常停靠。

在当前项目里有类似写法：

```xml
<DockPanel LastChildFill="True">
    <StackPanel DockPanel.Dock="Left" Orientation="Horizontal"/>
    <TextBlock TextAlignment="Right"/>
</DockPanel>
```

这里的效果是：左侧 `StackPanel` 停靠到左边，后面的 `TextBlock` 作为最后一个子控件，占据剩余空间，所以它可以在右侧区域显示状态文字。

### 2026-06-07：Grid 里面的 ColumnDefinitions 和 ColumnSpacing 是什么意思，WPF 里面好像不是这样写的？

**问题**

Grid 里面的 `ColumnDefinitions` 和 `ColumnSpacing` 是什么意思，WPF 里面好像不是这样写的。

**知识点**

`Grid` 是表格型布局容器，可以把界面切成多行、多列。`ColumnDefinitions` 用来定义列，`RowDefinitions` 用来定义行。

在 Avalonia 里，可以用简写：

```xml
<Grid ColumnDefinitions="3*,2*" ColumnSpacing="16">
    <Border Grid.Column="0"/>
    <ScrollViewer Grid.Column="1"/>
</Grid>
```

这里的意思是：

- `ColumnDefinitions="3*,2*"`：定义两列，第一列占剩余宽度的 3 份，第二列占剩余宽度的 2 份。
- `ColumnSpacing="16"`：两列之间留 16 像素间距。
- `Grid.Column="0"`：控件放到第 0 列，也就是第一列。
- `Grid.Column="1"`：控件放到第 1 列，也就是第二列。

`*` 是比例单位，不是固定像素。比如 `3*,2*` 表示总共 5 份，第一列占 3 份，第二列占 2 份。

WPF 里也有 `Grid`、`ColumnDefinitions`、`RowDefinitions`，但常见写法是属性元素语法：

```xml
<Grid>
    <Grid.ColumnDefinitions>
        <ColumnDefinition Width="3*"/>
        <ColumnDefinition Width="2*"/>
    </Grid.ColumnDefinitions>

    <Border Grid.Column="0"/>
    <ScrollViewer Grid.Column="1"/>
</Grid>
```

Avalonia 的 `ColumnDefinitions="3*,2*"` 是更短的写法，效果相当于 WPF 里写两个 `ColumnDefinition`。

另一个区别是：Avalonia 的 `Grid` 有 `ColumnSpacing` 和 `RowSpacing`，可以直接设置列间距、行间距。WPF 的原生 `Grid` 通常没有这个属性，如果要做列间距，常见做法是给子控件设置 `Margin`，或者额外插入一列作为间隔。

例如 WPF 里常见的间距写法：

```xml
<Grid>
    <Grid.ColumnDefinitions>
        <ColumnDefinition Width="3*"/>
        <ColumnDefinition Width="16"/>
        <ColumnDefinition Width="2*"/>
    </Grid.ColumnDefinitions>

    <Border Grid.Column="0"/>
    <ScrollViewer Grid.Column="2"/>
</Grid>
```

所以可以这样记：

`ColumnDefinitions` 负责“分几列、每列多宽”，`ColumnSpacing` 负责“列和列之间隔多远”。

### 2026-06-07：DataTemplate 是干什么的，为什么最后有很多一块一块的数据？

**问题**

下面这段 AXAML 里，`DataTemplate` 是干什么的？为什么最后界面上有很多一块一块的数据？这些块是怎么添加的？每一块的数据是怎么来的？

```xml
<ItemsControl ItemsSource="{Binding RecognizedTiles}">
    <ItemsControl.ItemsPanel>
        <ItemsPanelTemplate>
            <WrapPanel/>
        </ItemsPanelTemplate>
    </ItemsControl.ItemsPanel>
    <ItemsControl.ItemTemplate>
        <DataTemplate>
            <Border Width="86">
                <StackPanel>
                    <TextBlock Text="{Binding DisplayName}"/>
                    <TextBlock Text="{Binding Code}"/>
                    <TextBlock Text="{Binding ConfidenceText}"/>
                </StackPanel>
            </Border>
        </DataTemplate>
    </ItemsControl.ItemTemplate>
</ItemsControl>
```

**知识点**

`DataTemplate` 是“数据模板”，用来告诉 `ItemsControl`：集合里的每一条数据应该长成什么样。

`ItemsControl` 本身只知道要显示一组数据，不知道每条数据应该用按钮、文字、边框还是图片来显示。`DataTemplate` 就是给它的“单项显示模板”。

在这段代码里：

```xml
<ItemsControl ItemsSource="{Binding RecognizedTiles}">
```

`ItemsSource` 绑定到 ViewModel 里的 `RecognizedTiles` 集合。这个集合里每一个元素都是一张识别出的麻将牌，例如：

```csharp
new RecognizedMahjongTile("2m", "二万", 0.99)
new RecognizedMahjongTile("3m", "三万", 0.98)
new RecognizedMahjongTile("4m", "四万", 0.98)
```

`ItemsControl` 会遍历 `RecognizedTiles`。集合里有 13 个元素，就生成 13 个界面块。

每个界面块都使用同一个 `DataTemplate`：

```xml
<DataTemplate>
    <Border Width="86"
            Margin="0,0,8,8"
            Padding="8"
            BorderBrush="#D0D5DD"
            BorderThickness="1"
            Background="White"
            CornerRadius="4">
        <StackPanel Spacing="2">
            <TextBlock Text="{Binding DisplayName}"/>
            <TextBlock Text="{Binding Code}"/>
            <TextBlock Text="{Binding ConfidenceText}"/>
        </StackPanel>
    </Border>
</DataTemplate>
```

所以看到的“一块一块的数据”，其实就是这个 `Border` 被重复生成了很多次。

每一块的数据来自当前集合项。也就是说，在 `DataTemplate` 内部，`DataContext` 不再是整个 `MainWindowViewModel`，而是当前那一张牌 `RecognizedMahjongTile`。

因此：

- `{Binding DisplayName}` 绑定当前牌的显示名，例如“二万”。
- `{Binding Code}` 绑定当前牌的编码，例如 `2m`。
- `{Binding ConfidenceText}` 绑定当前牌的识别置信度文本，例如 `99%`。

这些数据是在 ViewModel 里添加进去的：

```csharp
RecognizedTiles.Clear();
foreach (var tile in recognitionResult.Tiles)
{
    RecognizedTiles.Add(tile);
}
```

`RecognizedTiles` 是 `ObservableCollection<RecognizedMahjongTile>`。它的特点是：添加、删除、清空元素时，会通知界面自动刷新。因此代码里 `RecognizedTiles.Add(tile)` 执行一次，界面上就会多生成一个对应的模板块。

`WrapPanel` 决定这些块怎么排列：

```xml
<ItemsPanelTemplate>
    <WrapPanel/>
</ItemsPanelTemplate>
```

它的效果是从左到右排列，宽度不够时自动换行。真正决定“每一块长什么样”的是 `DataTemplate`，决定“这些块怎么排”的是 `ItemsPanelTemplate` 里的 `WrapPanel`。

可以这样记：

`ItemsSource` 负责提供数据集合，`DataTemplate` 负责定义每条数据的界面样子，`ItemsPanelTemplate` 负责定义这些条目怎么排列。
